# Clawlova

A chat UI ("cockpit") for the [OpenClaw](https://openclaw.ai) AI agent gateway. pnpm monorepo with a single package at `packages/cockpit`.

## Development

```bash
pnpm install                          # install dependencies
pnpm dev                              # cockpit dev server on :3000 (needs gateway running)
pnpm test --filter=cockpit            # run tests
pnpm openclaw:ws -- "hello"           # test WebSocket chat from CLI
```

## Docker Compose

```bash
docker compose up --build             # start the full stack
```

Services:

| Service | Role |
|---------|------|
| `openclaw-init` | One-shot: generates config + bootstrap gateway token |
| `openclaw-gateway` | AI agent runtime (WebSocket on :18789) |
| `cockpit-bootstrap` | One-shot: generates device keypair, pairs with gateway |
| `cockpit` | Chat web app (port 3000) |
| `openclaw-cli` | Optional: interactive CLI for manual OpenClaw commands |

First startup takes ~60s while OpenClaw initializes and the device is paired.

## Environment Variables

At least one LLM provider API key is required.

| Variable | Default | Description |
|----------|---------|-------------|
| `OPENAI_API_KEY` | — | OpenAI API key |
| `ANTHROPIC_API_KEY` | — | Anthropic API key |
| `GOOGLE_API_KEY` | — | Google API key |
| `COCKPIT_PORT` | `3000` | Cockpit web UI port |
| `OPENCLAW_GATEWAY_PORT` | `18789` | Gateway port |

## Gotchas

- **File sending to OpenClaw:** use filesystem paths, not base64 encoding. Base64 does not work with the OpenClaw bridge.
```

```markdown
# Cockpit

## Tech Stack

React 19, TanStack Start (router + SSR), TanStack Query, Nitro server, Vite 7, Tailwind CSS v4, Zustand, Motion, TypeScript strict mode.

## Code Style

- No semicolons, single quotes, trailing commas (Prettier enforced)
- `pnpm --filter cockpit check` — auto-fix formatting + lint
- ESLint uses `@tanstack/eslint-config`
- React Compiler enabled via babel plugin — do not add manual `useMemo`/`useCallback`

## Path Aliases

`#/*` and `@/*` both resolve to `./src/*`. Prefer `#/*` — this is the convention used in existing code (defined via the package.json `imports` field).

## Generated Files

- `src/routeTree.gen.ts` — auto-generated by TanStack Router. Never edit manually.

## Project Structure

```
src/
  components/
    chat/           Chat UI (message bubbles, composer, tool cards, thinking blocks)
    config/         Config dashboard sections
    ui/             Shared UI primitives
  lib/
    chat-store.ts          Zustand store for chat state
    openclaw-bridge/       WebSocket bridge to OpenClaw gateway (auth, streaming,
                           message translation). Understand before modifying —
                           this is the critical data path.
    openclaw-sessions.ts   Session persistence + loading
    openclaw-config.ts     Gateway config read/write
    file-registry.ts       Uploaded file metadata
    utils.ts               Shared utilities
  integrations/
    tanstack-query/  TanStack Query provider + devtools
  routes/            File-based routes + API routes (api.chat.ts, api.file.ts, api.upload.ts)
  server/
    functions.ts     All server functions via createServerFn (TanStack Start)
```

## Architecture

- Server functions go through `createServerFn` in `src/server/functions.ts`
- Chat state: Zustand store at `src/lib/chat-store.ts`
- Data flow: Browser <-> Cockpit (SSE) <-> OpenClaw Gateway (WebSocket)
- Markdown: `react-markdown` + `remark-gfm` + `rehype-highlight`
- Icons: `lucide-react`
- Animations: `motion` library

## Testing

- Vitest with jsdom
- Tests colocated with source (e.g., `message-preview.test.ts` next to `message-preview.ts`)
- Run: `pnpm test --filter=cockpit`
```
