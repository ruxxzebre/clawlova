# CLAUDE.md Design Spec

## Goal

Create a layered CLAUDE.md setup for the Clawlova project: a root-level file for project-wide context and a cockpit-level file for package-specific conventions. Claude Code automatically merges both when working inside `packages/cockpit/`.

## Files

### 1. `/CLAUDE.md` — Project-wide

**Project overview:**
- Clawlova is a chat UI ("cockpit") for the OpenClaw AI agent gateway
- pnpm monorepo with a single package at `packages/cockpit`

**Development commands:**
- `pnpm install` — install dependencies
- `pnpm dev` — run cockpit dev server on port 3000 (requires OpenClaw gateway running)
- `pnpm test --filter=cockpit` — run tests
- `pnpm openclaw:ws -- "message"` — test WebSocket chat from CLI

**Docker Compose stack:**
- `docker compose up --build` starts the full stack
- Services: `openclaw-init` (one-shot config), `openclaw-gateway` (AI runtime on :18789), `cockpit-bootstrap` (device pairing), `cockpit` (web UI on :3000), `openclaw-cli` (optional)
- First startup takes ~60s for initialization + device pairing

**Environment variables:**
- At least one LLM provider key required: `OPENAI_API_KEY`, `ANTHROPIC_API_KEY`, or `GOOGLE_API_KEY`
- Key ports: `COCKPIT_PORT` (default 3000), `OPENCLAW_GATEWAY_PORT` (default 18789)

**Gotchas:**
- Image/file sending to OpenClaw: use filesystem paths, not base64 encoding (base64 doesn't work with the OpenClaw bridge)

### 2. `/packages/cockpit/CLAUDE.md` — Cockpit-specific

**Tech stack:**
- React 19, TanStack Start (router + SSR), TanStack Query, Nitro server, Vite 7
- Tailwind CSS v4, Zustand, Motion (framer-motion successor), TypeScript strict mode

**Code style:**
- No semicolons, single quotes, trailing commas (Prettier enforced)
- `pnpm --filter cockpit check` to auto-fix formatting + lint
- ESLint uses `@tanstack/eslint-config`
- React Compiler enabled via babel plugin — no need for manual `useMemo`/`useCallback`

**Path aliases:**
- `#/*` and `@/*` both map to `./src/*`
- Prefer `#/*` (convention used in existing code, defined via package.json `imports` field)

**Generated files:**
- `src/routeTree.gen.ts` is auto-generated by TanStack Router — never edit manually

**Project structure:**
- `src/components/chat/` — chat UI components (message bubbles, composer, tool cards, thinking blocks)
- `src/components/config/` — config dashboard sections
- `src/components/ui/` — shared UI primitives
- `src/lib/` — core logic (chat store, OpenClaw bridge, session management, utilities)
- `src/lib/openclaw-bridge/` — WebSocket bridge to OpenClaw gateway (auth, streaming, message translation). Understand before modifying — this is the critical data path.
- `src/routes/` — TanStack Router file-based routes + API routes (`api.chat.ts`, `api.file.ts`, `api.upload.ts`)
- `src/server/functions.ts` — all server functions use `createServerFn` from TanStack Start
- `src/data/` — TanStack Query integration

**Architecture patterns:**
- Server functions go through `createServerFn` in `src/server/functions.ts`
- Chat state managed by Zustand store (`src/lib/chat-store.ts`)
- Browser <-> Cockpit Server communication via SSE; Cockpit Server <-> OpenClaw Gateway via WebSocket
- Markdown rendering uses `react-markdown` + `remark-gfm` + `rehype-highlight`
- Icons from `lucide-react`
- Animations use `motion` library

**Testing:**
- Vitest with jsdom environment
- Tests colocated with source (e.g., `message-preview.test.ts` next to `message-preview.ts`)
- Run: `pnpm test --filter=cockpit`
